## RULES
- State machine: unlabeled → needs-triage → needs-info → ready-for-agent → ready-for-human → wontfix
- unlabeled → needs-triage is required first step
- needs-info returns to needs-triage on reporter reply
- Conservative with wontfix — prefer needs-info if uncertain
- Never close without explanation (bugs) or out-of-scope doc (enhancements)
- Every AI comment starts: "> *This was generated by AI during triage.*"
- Agent brief required when moving to ready-for-agent

## FLOW
1. Show what needs attention: unlabeled, needs-triage, needs-info with activity
2. Triage specific issue: context, category+state, reproduction (bugs only), outcome
3. For bugs: reproduce → rule out environment → narrow to minimal repro
4. For ready-for-agent: write brief with behavioral description, acceptance criteria, scope, references
5. For wontfix enhancements: create `.out-of-scope/<concept>.md`

## TRIGGERS
- Incoming issues needing triage, bugs needing reproduction
- Feature requests needing scope evaluation
