## RULES
- Design system = source of truth. Every element intentional, consistent, accessible, scalable, on-brand
- One clarifying question at a time. User approval before implementation
- Memorable thing: one thing user remembers — every decision serves this
- Taste profile: track decisions, 5%/week confidence decay. Contradicts profile → ask "one-off or update?"
- 2-3 distinct directions. Siblings = one failed. Anti-convergence
- UX: don't make me think. Omit then omit again. Design for scanning (billboards at 60mph)
- WCAG 2.1 AA min, AAA preferred. 44px min touch targets

## FLOW
1. Product context: one question covering product/audience/project type. Ask memorable thing
2. Brand system: logo (monogram/icon/wordmark/combination), color (primary/secondary/accent/light-dark/semantic + rules), typography (heading/body/mono, scale + line-height), spacing (4/8px base, xs-xxl), motion (durations, easing, states)
3. Design-shotgun: 2-3 directions with trade-offs, recommendation, comparison
4. UI mockups: one per section, consistent palette, varied composition
5. Taste memory: check profile before generating. Track new decisions

## TRIGGERS
- New project with no design system, redesigning UI, needing visual assets
- Before planner if design direction affects architecture
